* Reel Module
:PROPERTIES:
:CUSTOM_ID: reel-module
:END:
The Reel module is a declarative, programmatic video-creation framework built on
top of the engine's CPU image rasterizer.  You describe a composition as a
stack of timed, animated layers (clips), configure how time flows through each
layer using sequences, then drive a renderer to produce individual frames that
are streamed to an exporter (animated GIF, numbered image sequence, or MP4 via
ffmpeg).  The entire render pipeline is headless — no GPU, no display, no game
loop — making it suitable for CI asset baking, trailer generation, and offline
procedural video.

** Conceptual model
:PROPERTIES:
:CUSTOM_ID: conceptual-model
:END:
| Concept              | Type                              | Role                                                                           |
|----------------------+-----------------------------------+--------------------------------------------------------------------------------|
| Composition          | =LrgReel=                         | Registers id, resolution, fps, total duration, and the ordered clip stack      |
| Current frame        | =LrgReelContext=                  | Carries absolute frame + offset stack; clips read =get_frame()= from it        |
| Timed layer          | =LrgReelClip=                     | Draws itself onto a canvas for the current relative frame                      |
| Content clips        | =LrgReelSolidClip= et al.         | Ready-made clips: solid, gradient, image, text, shape, caption, video          |
| Time remapping       | =LrgReelSequence=                 | Shifts, sequences, loops, or freezes time for a group of child clips           |
| Clip chain           | =LrgReelTransitionSeries=         | Auto-sequences segments with overlapping transitions                           |
| Animation verbs      | =lrg_reel_interpolate= / spring   | Pure functions mapping a frame number to a value (color, noise, bezier too)    |
| Per-clip filter      | =LrgReelEffect=                   | Post-draw image filter: blur, bloom, grade, vignette, grain, key, shadow, leak |
| Frame engine         | =LrgReelRenderer=                 | Clears the canvas, composites active clips, motion blur, parallel render       |
| GPU capture          | =LrgReelGpuRenderer=              | GL framebuffer capture for engine-rendered scenes                              |
| Output sink          | =LrgReelExporter=                 | Abstract: GIF / image-sequence / MP4 / audio-only concrete subclasses          |
| CPU crossfade        | =LrgReelTransition=               | Blends two =GrlImage= frames at a progress value (9 types)                     |
| Offline audio mix    | =LrgReelAudioTrack=               | Places timed wave clips on a timeline and mixes to =LrgWaveData=               |
| Video source         | =LrgReelVideoSource=              | Decodes an existing video file; frames available on demand                     |
| Audio analysis       | =lrg_reel_audio_spectrum= et al.  | Per-frame FFT spectrum, waveform, and RMS level                                |
| Data-driven          | =lrg_reel_load_yaml=              | Builds an =LrgReel= from a YAML description file                               |
| CLI                  | =reel= binary                     | info / still / render subcommands with codec and thread options                |
| Live preview         | =LrgReelPlayer=                   | Opens a GPU window for interactive playback; the only windowed component       |

** Quick start
:PROPERTIES:
:CUSTOM_ID: quick-start
:END:
The following is the complete =reel-hello.c= example.  It renders a 2-second
title card to an animated GIF without touching a display.

#+begin_src c
#include <libregnum.h>
#include <graylib.h>

static void
draw_title (LrgReelClip    *clip,
            LrgReelContext *ctx,
            LrgImageCanvas *canvas,
            gpointer        user_data)
{
    gint     frame = lrg_reel_context_get_frame (ctx);
    gint     width = lrg_reel_context_get_width (ctx);
    GrlColor bg = { 12, 14, 28, 255 };
    GrlColor fg = { 255, 255, 255, 255 };
    gdouble  opacity;
    gdouble  y;

    /* Background. */
    lrg_image_canvas_clear (canvas, &bg);

    /* Fade the text in over the first 20 frames. */
    opacity = lrg_reel_interpolate_clamped (frame, 0.0, 20.0, 0.0, 1.0,
                                            LRG_EASING_EASE_OUT_CUBIC);

    /* Spring the baseline up from below into place. */
    y = lrg_reel_spring (frame, 30.0, NULL, 220.0, 150.0);

    fg.a = (guint8) (opacity * 255.0);
    grl_image_draw_text_bitmap (lrg_image_canvas_get_image (canvas),
                                "HELLO REEL",
                                width / 2 - 110, (gint) y, 32, &fg);
}

int
main (void)
{
    g_autoptr(LrgReel)            reel = NULL;
    g_autoptr(LrgReelClip)        title = NULL;
    g_autoptr(LrgReelRenderer)    renderer = NULL;
    g_autoptr(LrgReelGifExporter) gif = NULL;
    g_autoptr(GError)             error = NULL;

    reel = lrg_reel_new ("hello", 640, 360, 30.0, 60);

    title = lrg_reel_clip_new_with_func (draw_title, NULL, NULL);
    lrg_reel_add_clip (reel, title);

    renderer = lrg_reel_renderer_new (reel);

    gif = lrg_reel_gif_exporter_new ("reel-hello.gif", &error);
    if (gif == NULL)
    {
        g_printerr ("Failed to open output: %s\n", error->message);
        return 1;
    }

    if (!lrg_reel_renderer_render_to_exporter (renderer,
                                               LRG_REEL_EXPORTER (gif),
                                               &error))
    {
        g_printerr ("Render failed: %s\n", error->message);
        return 1;
    }

    g_print ("Wrote reel-hello.gif (%d frames)\n",
             lrg_reel_get_duration_in_frames (reel));
    return 0;
}
#+end_src

Key points:
- =lrg_reel_new= fixes the composition's identity and dimensions up front.
- A render callback is the simplest way to author a clip; no subclassing needed.
- =lrg_reel_context_get_frame()= gives the relative (clip-local) frame.
- =lrg_reel_renderer_render_to_exporter()= drives the full render loop in one call.
- The GIF exporter requires no external tools; everything runs in-process.
