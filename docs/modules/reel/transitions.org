* Transitions — CPU crossfades
:PROPERTIES:
:CUSTOM_ID: transitions-cpu-crossfades
:END:
=LrgReelTransition= and its four concrete subclasses blend two =GrlImage= frames
together using the CPU rasterizer.  All transitions are headless and designed
for offline frame baking; no GPU or display is required.

Transitions are not invoked automatically by the renderer.  They are tools you
call manually inside a clip's render callback or in a custom render loop when
you want a timed crossfade between two pre-rendered frames.

** The contract: progress 0 → from, 1 → to
:PROPERTIES:
:CUSTOM_ID: progress-contract
:END:
The single invariant shared by all transition types:

- At =progress = 0.0= the canvas shows =from= entirely.
- At =progress = 1.0= the canvas shows =to= entirely.
- Between 0 and 1 the visual interpolation is transition-specific.

The public entry-point =lrg_reel_transition_composite()= clamps =progress= to
=[0, 1]=, applies the configured easing curve, and then calls the =composite=
vfunc with the eased value.

#+begin_src c
void
lrg_reel_transition_composite (LrgReelTransition *self,
                                LrgImageCanvas    *canvas,
                                GrlImage          *from,
                                GrlImage          *to,
                                gdouble            progress);
#+end_src

=canvas=, =from=, and =to= must all share the same pixel dimensions.

** Easing
:PROPERTIES:
:CUSTOM_ID: easing
:END:
The transition applies an =LrgEasingType= to the raw =progress= value before
dispatching to the composite vfunc.  The default is =LRG_EASING_LINEAR=.

#+begin_src c
lrg_reel_transition_set_easing (transition, LRG_EASING_EASE_IN_OUT_CUBIC);
LrgEasingType e = lrg_reel_transition_get_easing (transition);
#+end_src

** LrgReelFadeTransition — cross-fade
:PROPERTIES:
:CUSTOM_ID: lrgreelfadetransition
:END:
The outgoing frame is drawn fully onto the canvas; the incoming frame is then
composited over it at opacity equal to the eased progress.

#+begin_src c
g_autoptr(LrgReelFadeTransition) fade = lrg_reel_fade_transition_new ();
lrg_reel_transition_set_easing (LRG_REEL_TRANSITION (fade),
                                 LRG_EASING_EASE_IN_OUT_SINE);
lrg_reel_transition_composite (LRG_REEL_TRANSITION (fade),
                                canvas, frame_a, frame_b, progress);
#+end_src

** LrgReelWipeTransition — hard-edge wipe
:PROPERTIES:
:CUSTOM_ID: lrgreelwipetransition
:END:
A rectangular reveal grows from one edge, progressively replacing =from= with
=to=.

| Direction value                         | Reveal grows from ... |
|-----------------------------------------+-----------------------|
| =LRG_REEL_TRANSITION_DIRECTION_RIGHT=   | Left edge rightward   |
| =LRG_REEL_TRANSITION_DIRECTION_LEFT=    | Right edge leftward   |
| =LRG_REEL_TRANSITION_DIRECTION_DOWN=    | Top edge downward     |
| =LRG_REEL_TRANSITION_DIRECTION_UP=      | Bottom edge upward    |

#+begin_src c
g_autoptr(LrgReelWipeTransition) wipe =
    lrg_reel_wipe_transition_new (LRG_REEL_TRANSITION_DIRECTION_RIGHT);

/* Change direction after construction. */
lrg_reel_wipe_transition_set_direction (wipe,
    LRG_REEL_TRANSITION_DIRECTION_DOWN);

lrg_reel_transition_composite (LRG_REEL_TRANSITION (wipe),
                                canvas, from, to, progress);
#+end_src

** LrgReelDissolveTransition — per-pixel dissolve
:PROPERTIES:
:CUSTOM_ID: lrgreeldissolvetransition
:END:
Each pixel switches from =from= to =to= at a threshold determined by a
deterministic hash of the pixel coordinates and a seed value.  The reveal order
looks random but is perfectly reproducible for a given seed — important for
offline rendering where frames must be identical across runs.

The algorithm is O(width × height) per composite call.

#+begin_src c
g_autoptr(LrgReelDissolveTransition) dissolve =
    lrg_reel_dissolve_transition_new ();  /* default seed: 1337 */

/* Change seed for a different but equally reproducible pattern. */
lrg_reel_dissolve_transition_set_seed (dissolve, 42);
guint seed = lrg_reel_dissolve_transition_get_seed (dissolve);

lrg_reel_transition_composite (LRG_REEL_TRANSITION (dissolve),
                                canvas, from, to, progress);
#+end_src

** LrgReelSlideTransition — slide-in
:PROPERTIES:
:CUSTOM_ID: lrgreelslidetransition
:END:
The =from= frame stays stationary; =to= slides in from the edge indicated by
the direction.  At =progress = 0= the incoming frame is fully off-screen; at
=progress = 1= it covers the canvas.

| Direction value                         | Incoming frame enters from ... |
|-----------------------------------------+--------------------------------|
| =LRG_REEL_TRANSITION_DIRECTION_LEFT=    | Left edge, slides rightward    |
| =LRG_REEL_TRANSITION_DIRECTION_RIGHT=   | Right edge, slides leftward    |
| =LRG_REEL_TRANSITION_DIRECTION_UP=      | Top edge, slides downward      |
| =LRG_REEL_TRANSITION_DIRECTION_DOWN=    | Bottom edge, slides upward     |

#+begin_src c
g_autoptr(LrgReelSlideTransition) slide =
    lrg_reel_slide_transition_new (LRG_REEL_TRANSITION_DIRECTION_LEFT);

lrg_reel_transition_composite (LRG_REEL_TRANSITION (slide),
                                canvas, from, to, progress);
#+end_src

** Using a transition inside a clip
:PROPERTIES:
:CUSTOM_ID: using-transition-inside-clip
:END:
A typical pattern is to render =from= and =to= each frame separately, then
blend with a transition at a progress derived from the current frame:

#+begin_src c
static void
draw_crossfade (LrgReelClip    *clip,
                LrgReelContext *ctx,
                LrgImageCanvas *canvas,
                gpointer        user_data)
{
    MyData *data = (MyData *) user_data;
    gint    frame = lrg_reel_context_get_frame (ctx);
    gdouble progress = lrg_reel_interpolate_clamped (frame, 0.0, 30.0,
                                                      0.0, 1.0,
                                                      LRG_EASING_LINEAR);
    /* Render source and target frames into temporary images. */
    g_autoptr(GrlImage) from_img =
        lrg_reel_renderer_render_frame (data->from_renderer, 0);
    g_autoptr(GrlImage) to_img =
        lrg_reel_renderer_render_frame (data->to_renderer, 0);

    lrg_reel_transition_composite (LRG_REEL_TRANSITION (data->fade),
                                    canvas, from_img, to_img, progress);
}
#+end_src

** Custom transitions
:PROPERTIES:
:CUSTOM_ID: custom-transitions
:END:
Subclass =LrgReelTransition= and override the =composite= vfunc to implement
any CPU-based effect:

#+begin_src c
struct _MyIrisTransitionClass
{
    LrgReelTransitionClass parent_class;
};

static void
my_iris_composite (LrgReelTransition *self,
                   LrgImageCanvas    *canvas,
                   GrlImage          *from,
                   GrlImage          *to,
                   gdouble            progress)
{
    /* progress is already eased by the base class. */
    /* ... circular reveal using grl_image_* draw calls ... */
}

static void
my_iris_transition_class_init (MyIrisTransitionClass *klass)
{
    LRG_REEL_TRANSITION_CLASS (klass)->composite = my_iris_composite;
}
#+end_src

The =composite= vfunc receives the *eased* progress; the base class has already
applied the easing curve before dispatching.

** LrgReelFlipTransition — card flip
:PROPERTIES:
:CUSTOM_ID: lrgreelfliptransition
:END:
Simulates a flat card rotating about its central axis.  In the first half of
the transition the outgoing frame shrinks on the flip axis to simulate the
card turning away; in the second half the incoming frame grows back in.

| Direction value                         | Axis of rotation       |
|-----------------------------------------+------------------------|
| =LRG_REEL_TRANSITION_DIRECTION_LEFT=    | Horizontal (Y axis)    |
| =LRG_REEL_TRANSITION_DIRECTION_RIGHT=   | Horizontal (Y axis)    |
| =LRG_REEL_TRANSITION_DIRECTION_UP=      | Vertical (X axis)      |
| =LRG_REEL_TRANSITION_DIRECTION_DOWN=    | Vertical (X axis)      |

#+begin_src c
g_autoptr(LrgReelFlipTransition) flip =
    lrg_reel_flip_transition_new (LRG_REEL_TRANSITION_DIRECTION_LEFT);

lrg_reel_transition_composite (LRG_REEL_TRANSITION (flip),
                                canvas, from, to, progress);
#+end_src

** LrgReelClockWipeTransition — angular sweep
:PROPERTIES:
:CUSTOM_ID: lrgreelclockwipetransition
:END:
Reveals the incoming frame through a clockwise angular sweep starting at the
12 o'clock position.  The sweep is per-pixel: a pixel at angle θ (normalised
to [0, 1) clockwise from straight up) switches from =from= to =to= when the
eased progress exceeds θ.

#+begin_src c
g_autoptr(LrgReelClockWipeTransition) cw =
    lrg_reel_clock_wipe_transition_new ();

lrg_reel_transition_set_easing (LRG_REEL_TRANSITION (cw),
                                 LRG_EASING_EASE_IN_OUT_QUAD);
lrg_reel_transition_composite (LRG_REEL_TRANSITION (cw),
                                canvas, from, to, progress);
#+end_src

** LrgReelIrisTransition — circular iris
:PROPERTIES:
:CUSTOM_ID: lrgreeliristry
:END:
Reveals the incoming frame through a circle that grows outward from the frame
centre.  Set =inverse = TRUE= to run the effect in reverse (a circle of the
outgoing frame closes inward instead).

#+begin_src c
g_autoptr(LrgReelIrisTransition) iris =
    lrg_reel_iris_transition_new ();

/* Close instead of open — useful for ending a scene. */
lrg_reel_iris_transition_set_inverse (iris, TRUE);

lrg_reel_transition_composite (LRG_REEL_TRANSITION (iris),
                                canvas, from, to, progress);
#+end_src

** LrgReelPushTransition — simultaneous push
:PROPERTIES:
:CUSTOM_ID: lrgreelpushtransition
:END:
Both frames move together: =from= slides out of the canvas while =to= slides in
from the opposite edge, maintaining a seamless contact line between them.  This
differs from the slide transition where only the incoming frame moves.

| Direction value                         | from exits toward ...  | to enters from ... |
|-----------------------------------------+------------------------+--------------------|
| =LRG_REEL_TRANSITION_DIRECTION_LEFT=    | Left                   | Right              |
| =LRG_REEL_TRANSITION_DIRECTION_RIGHT=   | Right                  | Left               |
| =LRG_REEL_TRANSITION_DIRECTION_UP=      | Top                    | Bottom             |
| =LRG_REEL_TRANSITION_DIRECTION_DOWN=    | Bottom                 | Top                |

#+begin_src c
g_autoptr(LrgReelPushTransition) push =
    lrg_reel_push_transition_new (LRG_REEL_TRANSITION_DIRECTION_LEFT);

lrg_reel_transition_composite (LRG_REEL_TRANSITION (push),
                                canvas, from, to, progress);
#+end_src

** LrgReelZoomTransition — centre zoom
:PROPERTIES:
:CUSTOM_ID: lrgreelzoomtransition
:END:
The incoming frame zooms in from 30 % of the frame size while simultaneously
fading in over the outgoing frame.  At =progress = 0= the incoming frame is
scaled to 30 % and fully transparent; at =progress = 1= it is at full size and
full opacity.

#+begin_src c
g_autoptr(LrgReelZoomTransition) zoom =
    lrg_reel_zoom_transition_new ();

lrg_reel_transition_set_easing (LRG_REEL_TRANSITION (zoom),
                                 LRG_EASING_EASE_OUT_CUBIC);
lrg_reel_transition_composite (LRG_REEL_TRANSITION (zoom),
                                canvas, from, to, progress);
#+end_src

** LrgReelTransitionSeries — auto-sequenced clip chain
:PROPERTIES:
:CUSTOM_ID: lrgreeltransitionseries
:END:
=LrgReelTransitionSeries= is itself an =LrgReelClip= that chains a sequence of
segments with optional overlapping transitions between them.  It computes a
unified timeline automatically: each transition overlaps the tail of the
outgoing segment and the head of the incoming segment, shrinking the total
duration by the transition length.

#+begin_src c
g_autoptr(LrgReelTransitionSeries) series =
    lrg_reel_transition_series_new ();

/* Segment A — 60 frames. */
lrg_reel_transition_series_add (series,
                                LRG_REEL_CLIP (clip_a),
                                60);

/* 15-frame cross-fade into B. */
g_autoptr(LrgReelFadeTransition) fade = lrg_reel_fade_transition_new ();
lrg_reel_transition_series_add_transition (series,
                                           LRG_REEL_TRANSITION (fade),
                                           15);

/* Segment B — 90 frames. */
lrg_reel_transition_series_add (series,
                                LRG_REEL_CLIP (clip_b),
                                90);

/* 20-frame push into C. */
g_autoptr(LrgReelPushTransition) push =
    lrg_reel_push_transition_new (LRG_REEL_TRANSITION_DIRECTION_LEFT);
lrg_reel_transition_series_add_transition (series,
                                           LRG_REEL_TRANSITION (push),
                                           20);

/* Segment C — 45 frames. */
lrg_reel_transition_series_add (series,
                                LRG_REEL_CLIP (clip_c),
                                45);

/* Total = 60 + 90 + 45 - 15 - 20 = 160 frames. */
guint total = lrg_reel_transition_series_get_total_frames (series);

lrg_reel_add_clip (reel, LRG_REEL_CLIP (series));
#+end_src

The series duration is also automatically written back to the clip's own
=duration-in-frames= property, so parent containers see the correct length.

*** Construction order

Items must alternate: segment, optional transition, segment, optional
transition, …  Calling =add_transition()= twice in a row (without an =add()=
between them) is undefined.  Calling =add()= twice in a row creates a hard
cut (no transition).

** Pitfalls
:PROPERTIES:
:CUSTOM_ID: pitfalls
:END:
- All transitions are CPU-side operations on =GrlImage= data.  They can be slow
  on large frame sizes (e.g. 4K); profile if frame rate during export is a
  concern.
- =LrgReelDissolveTransition= is O(width × height) per call.  For large
  resolutions prefer =LrgReelFadeTransition= if performance is critical.
- The =composite= vfunc receives the *eased* progress value.  If you override
  =composite= in a subclass, do not apply easing a second time inside the vfunc.
- Transition direction semantics differ between wipe and slide: for wipe,
  =DIRECTION_RIGHT= means the reveal grows rightward (from the left edge); for
  slide, =DIRECTION_RIGHT= means the incoming frame enters from the right edge
  (slides leftward into place).  Read the header comments if in doubt.
- =LrgReelTransitionSeries= takes references to both clip and transition objects;
  callers retain their own references and must not assume exclusive ownership.
- The =get_total_frames()= return value changes whenever =add()= or
  =add_transition()= is called.  Query it only after the series is fully
  constructed.
