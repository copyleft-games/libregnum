* Animation math — interpolate and spring
:PROPERTIES:
:CUSTOM_ID: animation-math
:END:
The Reel module ships two pure, stateless animation primitives designed for
frame-by-frame offline rendering.  Neither requires a manager, a game loop, or
any mutable state: you pass the current frame number and receive an output
value.

Both functions build on the engine's =LrgEasingType= library.

** lrg_reel_interpolate
:PROPERTIES:
:CUSTOM_ID: lrg-reel-interpolate
:END:
Maps an input value (typically the current frame) through a piecewise curve
defined by parallel input/output breakpoint arrays.

#+begin_src c
gdouble
lrg_reel_interpolate (gdouble             input,
                      const gdouble      *input_range,
                      gsize               n_input,
                      const gdouble      *output_range,
                      gsize               n_output,
                      LrgEasingType       easing,
                      LrgReelExtrapolate  extrapolate_left,
                      LrgReelExtrapolate  extrapolate_right);
#+end_src

=n_input= and =n_output= must be equal and at least 2.  Easing is applied
within each segment independently.

*** Example: multi-stop opacity keyframes
#+begin_src c
static void
my_render (LrgReelClip *clip, LrgReelContext *ctx,
           LrgImageCanvas *canvas, gpointer ud)
{
    gint    frame = lrg_reel_context_get_frame (ctx);
    /* Fade in 0->1 over frames 0..15, hold, fade out 1->0 over frames 45..60. */
    const gdouble in[]  = { 0.0, 15.0, 45.0, 60.0 };
    const gdouble out[] = { 0.0,  1.0,  1.0,  0.0 };
    gdouble opacity = lrg_reel_interpolate (
        (gdouble) frame,
        in,  G_N_ELEMENTS (in),
        out, G_N_ELEMENTS (out),
        LRG_EASING_EASE_IN_OUT_CUBIC,
        LRG_REEL_EXTRAPOLATE_CLAMP,
        LRG_REEL_EXTRAPOLATE_CLAMP);
    /* use opacity ... */
}
#+end_src

*** Extrapolate modes
:PROPERTIES:
:CUSTOM_ID: extrapolate-modes
:END:
| Enum value                       | Behaviour outside the input range                           |
|----------------------------------+-------------------------------------------------------------|
| =LRG_REEL_EXTRAPOLATE_CLAMP=     | Returns the first or last output value (most common)        |
| =LRG_REEL_EXTRAPOLATE_EXTEND=    | Linearly extends the first or last segment beyond the range |
| =LRG_REEL_EXTRAPOLATE_IDENTITY=  | Returns the input value unchanged                           |
| =LRG_REEL_EXTRAPOLATE_WRAP=      | Wraps the input modulo the range width                      |

=CLAMP= is the right default for nearly all animation uses.  =EXTEND= is useful
for a value that should continue growing after the last keyframe.  =IDENTITY=
passes the raw frame number through; avoid it unless you have a specific reason.
=WRAP= creates a seamlessly repeating pattern from any segment.

** lrg_reel_interpolate_clamped
:PROPERTIES:
:CUSTOM_ID: lrg-reel-interpolate-clamped
:END:
The convenience overload for the common two-point case.  Uses
=LRG_REEL_EXTRAPOLATE_CLAMP= on both sides automatically.

#+begin_src c
gdouble
lrg_reel_interpolate_clamped (gdouble       input,
                              gdouble       in_min,
                              gdouble       in_max,
                              gdouble       out_min,
                              gdouble       out_max,
                              LrgEasingType easing);
#+end_src

#+begin_src c
/* Fade opacity from 0 to 1 over the first 20 frames; clamp outside. */
gdouble opacity = lrg_reel_interpolate_clamped (
    (gdouble) frame,
    0.0, 20.0,
    0.0, 1.0,
    LRG_EASING_EASE_OUT_CUBIC);

/* Slide x from 640 to 320 over frames 10..40. */
gdouble x = lrg_reel_interpolate_clamped (
    (gdouble) frame,
    10.0, 40.0,
    640.0, 320.0,
    LRG_EASING_EASE_IN_OUT_BACK);
#+end_src

** lrg_reel_spring
:PROPERTIES:
:CUSTOM_ID: lrg-reel-spring
:END:
Evaluates a damped harmonic oscillator at a given frame, animating from =from=
to =to=.

#+begin_src c
gdouble
lrg_reel_spring (gint                       frame,
                 gdouble                    fps,
                 const LrgReelSpringConfig *config,
                 gdouble                    from,
                 gdouble                    to);
#+end_src

Pass =NULL= for =config= to use the sensible defaults (mass 1.0, stiffness 100.0,
damping 10.0, no initial velocity, no clamping):

#+begin_src c
/* Spring a title's y coordinate from 220 (below) to 150 (resting). */
gdouble y = lrg_reel_spring (frame, 30.0, NULL, 220.0, 150.0);
#+end_src

At frame 0 the value equals =from= exactly; it then oscillates toward =to=.
Underdamped configurations (low damping relative to stiffness) overshoot and
bounce; critically-damped and overdamped configurations approach =to= without
crossing it.

*** LrgReelSpringConfig
:PROPERTIES:
:CUSTOM_ID: lrgreelspringconfig
:END:
=LrgReelSpringConfig= is a *GBoxed value type*, not a GObject.  Allocate it with
=lrg_reel_spring_config_new()= and free it with =lrg_reel_spring_config_free()=
(or use =g_autoptr=).

#+begin_src c
g_autoptr(LrgReelSpringConfig) cfg = lrg_reel_spring_config_new ();
cfg->stiffness  = 200.0;   /* stiffer spring */
cfg->damping    = 12.0;    /* a little underdamped for a small bounce */
cfg->mass       = 1.0;
cfg->initial_velocity = 0.0;
cfg->overshoot_clamping = FALSE;

gdouble y = lrg_reel_spring (frame, 30.0, cfg, 300.0, 100.0);
#+end_src

| Field                 | Default | Meaning                                                        |
|-----------------------+---------+----------------------------------------------------------------|
| =mass=                | 1.0     | Oscillator inertia; higher mass moves slower (must be > 0)     |
| =stiffness=           | 100.0   | Spring constant; higher values snap faster (must be > 0)       |
| =damping=             | 10.0    | Friction coefficient; raise to reduce overshoot (must be >= 0) |
| =initial_velocity=    | 0.0     | Starting velocity in value-units per second                    |
| =overshoot_clamping=  | FALSE   | If TRUE the value never passes =to= even in underdamped cases  |

*** Sizing a sequence to a spring

When you need a sequence duration that exactly covers a spring animation:

#+begin_src c
/* How many frames until the spring settles (within a small threshold)? */
guint settle = lrg_reel_spring_duration_in_frames (30.0, cfg);
LrgReelSequence *seq = lrg_reel_sequence_new (0, (gint) settle);
#+end_src

*** Combining spring and interpolate

Spring and interpolate compose naturally in a single render callback:

#+begin_src c
/* y springs up; opacity fades in simultaneously. */
gdouble y       = lrg_reel_spring (frame, 30.0, NULL, 200.0, 100.0);
gdouble opacity = lrg_reel_interpolate_clamped (frame, 0.0, 20.0,
                                                0.0, 1.0,
                                                LRG_EASING_EASE_OUT_CUBIC);
#+end_src

** lrg_reel_interpolate_color (perceptual color blending)
:PROPERTIES:
:CUSTOM_ID: lrg-reel-interpolate-color
:END:
Like =lrg_reel_interpolate= but for =GrlColor= values.  Adjacent color stops are
blended in the OkLab perceptual color space so that the brightness and hue
perceived by the viewer vary evenly across the transition — avoiding the muddy,
desaturated mid-tones that occur when interpolating naively in linear RGB or
sRGB.

#+begin_src c
gdouble
lrg_reel_interpolate_color (gdouble             input,
                            const gdouble      *input_range,
                            gsize               n_input,
                            const GrlColor     *colors,
                            gsize               n_colors,
                            LrgEasingType       easing,
                            LrgReelExtrapolate  extrapolate_left,
                            LrgReelExtrapolate  extrapolate_right);
#+end_src

Returns a =(transfer full)= =GrlColor *=.  Free it with =grl_color_free()= or
use =g_autoptr(GrlColor)=.

*** Two-stop convenience variant

#+begin_src c
GrlColor *
lrg_reel_interpolate_color_clamped (gdouble         input,
                                    gdouble         in_min,
                                    gdouble         in_max,
                                    const GrlColor *color_a,
                                    const GrlColor *color_b,
                                    LrgEasingType   easing);
#+end_src

*** Example: animated sky gradient

#+begin_src c
static void
draw_sky (LrgReelClip    *clip,
          LrgReelContext *ctx,
          LrgImageCanvas *canvas,
          gpointer        user_data)
{
    gint frame = lrg_reel_context_get_frame (ctx);

    GrlColor dawn    = {  30,  20,  80, 255 };
    GrlColor sunrise = { 240, 120,  40, 255 };
    GrlColor noon    = {  80, 160, 240, 255 };

    const gdouble stops[] = {  0.0, 30.0, 90.0 };
    const GrlColor clrs[] = { dawn, sunrise, noon };

    g_autoptr(GrlColor) sky =
        lrg_reel_interpolate_color ((gdouble) frame,
                                    stops, G_N_ELEMENTS (stops),
                                    clrs,  G_N_ELEMENTS (clrs),
                                    LRG_EASING_EASE_IN_OUT_SINE,
                                    LRG_REEL_EXTRAPOLATE_CLAMP,
                                    LRG_REEL_EXTRAPOLATE_CLAMP);

    lrg_image_canvas_clear (canvas, sky);
}
#+end_src

*** Pitfall

=lrg_reel_interpolate_color= returns =(transfer full)=: you must free the
returned =GrlColor *=.  A common mistake is to compare it with a raw struct.
Always use =g_autoptr(GrlColor)= or call =grl_color_free()= explicitly.

** lrg_reel_easing_bezier — custom cubic-bezier easing
:PROPERTIES:
:CUSTOM_ID: lrg-reel-easing-bezier
:END:
Evaluates a CSS-style cubic-bezier easing curve, identical in semantics to
the CSS =cubic-bezier(x1,y1,x2,y2)= function.  This is useful when the named
=LrgEasingType= values are not precise enough.

#+begin_src c
gdouble
lrg_reel_easing_bezier (gdouble x1,
                        gdouble y1,
                        gdouble x2,
                        gdouble y2,
                        gdouble t);
#+end_src

=t= is a normalised progress in [0, 1].  The curve always passes through (0, 0)
and (1, 1).  =bezier(0, 0, 1, 1, t) == t= (linear).

#+begin_src c
/* A "snappy" ease-out similar to Material Design. */
gdouble eased = lrg_reel_easing_bezier (0.0, 0.0, 0.2, 1.0, raw_t);

/* Apply it to a frame-based interpolation. */
gdouble raw_t  = lrg_reel_interpolate_clamped ((gdouble) frame,
                                                0.0, 30.0,
                                                0.0, 1.0,
                                                LRG_EASING_LINEAR);
gdouble y = raw_t * 200.0;              /* linear position */
gdouble y_eased = lrg_reel_easing_bezier (0.0, 0.0, 0.2, 1.0, raw_t)
                  * 200.0;              /* snappy position */
#+end_src

*** Pitfall

The =x1= and =x2= control points must lie in [0, 1] (they represent time
fractions).  =y1= and =y2= have no such constraint; values outside [0, 1]
produce overshoot/undershoot — this is intentional for bounce effects.

** lrg_reel_random and lrg_reel_random_range — deterministic randomness
:PROPERTIES:
:CUSTOM_ID: lrg-reel-random
:END:
Deterministic pseudo-random values derived purely from a seed.  The same seed
always produces the same value, making renders reproducible across runs.  Pass
a compound key such as =(frame * 1000 + clip_index)= to get independent streams
per clip and per frame.

#+begin_src c
gdouble lrg_reel_random       (guint64 seed);
gdouble lrg_reel_random_range (guint64 seed, gdouble min, gdouble max);
#+end_src

#+begin_src c
static void
draw_starfield (LrgReelClip    *clip,
                LrgReelContext *ctx,
                LrgImageCanvas *canvas,
                gpointer        user_data)
{
    gint     frame = lrg_reel_context_get_frame (ctx);
    GrlImage *img  = lrg_image_canvas_get_image (canvas);

    for (gint i = 0; i < 200; i++)
    {
        /* Position is fixed per star (seed = i); brightness flickers with frame. */
        gdouble x = lrg_reel_random_range ((guint64) i * 2,       0.0, 640.0);
        gdouble y = lrg_reel_random_range ((guint64) i * 2 + 1,   0.0, 360.0);
        gdouble b = lrg_reel_random       ((guint64) frame * 300 + (guint64) i);

        guint8 bright = (guint8) (b * 255.0);
        GrlColor star = { bright, bright, bright, 255 };
        grl_image_draw_pixel (img, (gint) x, (gint) y, &star);
    }
}
#+end_src

** lrg_reel_noise_1d / 2d / 3d — smooth value noise
:PROPERTIES:
:CUSTOM_ID: lrg-reel-noise
:END:
Smooth, deterministic value noise in [-1.0, 1.0].  Adjacent sample coordinates
produce close results (the noise is continuous), making it ideal for organic
motion such as camera shake, wobble, or atmospheric haze.

#+begin_src c
gdouble lrg_reel_noise_1d (gdouble x);
gdouble lrg_reel_noise_2d (gdouble x, gdouble y);
gdouble lrg_reel_noise_3d (gdouble x, gdouble y, gdouble z);
#+end_src

#+begin_src c
static void
draw_wobble (LrgReelClip    *clip,
             LrgReelContext *ctx,
             LrgImageCanvas *canvas,
             gpointer        user_data)
{
    gint    frame = lrg_reel_context_get_frame (ctx);
    gdouble t     = frame / 30.0;    /* convert frame to time in seconds */

    /* Slow, organic X and Y jitter at different noise phases. */
    gdouble jitter_x = lrg_reel_noise_1d (t * 0.8)         * 12.0;
    gdouble jitter_y = lrg_reel_noise_1d (t * 0.8 + 100.0) * 12.0;

    lrg_reel_clip_set_x (clip, 320.0 + jitter_x);
    lrg_reel_clip_set_y (clip, 180.0 + jitter_y);
}
#+end_src

Use =noise_2d= when you need a spatial field (e.g. per-pixel distortion) and
=noise_3d= when you additionally want to animate the field over time by passing
the frame as the Z dimension.

** Pitfalls
:PROPERTIES:
:CUSTOM_ID: pitfalls
:END:
- =LrgReelSpringConfig= is a *GBoxed* type.  Never call =g_object_unref()= on
  it — use =lrg_reel_spring_config_free()= or =g_autoptr(LrgReelSpringConfig)=.
- Both functions accept =gdouble= input and output; cast your =gint= frame with
  =(gdouble) frame= before passing it.
- =lrg_reel_interpolate= asserts that =n_input == n_output >= 2= and that
  =input_range= is monotonically increasing.  Violating these will abort.
- =lrg_reel_spring_duration_in_frames= returns an estimate; do not rely on it
  for pixel-exact sync.  Verify visually with =LrgReelPlayer= if timing is
  critical.
- These are pure functions — calling them multiple times with the same arguments
  always returns the same value.  Do not cache results across frames of a loop
  sequence; call them fresh each frame with the current relative frame.
- =lrg_reel_interpolate_color= returns =(transfer full)=: free the returned
  =GrlColor *= with =grl_color_free()=.  Do not call =g_object_unref()= on it
  — =GrlColor= is a GBoxed type, not a GObject.
- Noise functions are deterministic for a given coordinate, so they do not
  change between render passes.  If you need per-run variation, offset the
  coordinate by a fixed seed that you embed in the composition.
