* Tween Module
:PROPERTIES:
:CUSTOM_ID: tween-module
:END:
The Tween module provides property animation with easing functions, sequences, and parallel execution for smooth UI animations, camera movements, and game object transitions.

** Overview
:PROPERTIES:
:CUSTOM_ID: overview
:END:
The tweening system follows a hierarchical pattern for flexible animation composition:

- *LrgEasing* - Library of 30+ easing functions (linear, quad, cubic, elastic, bounce, etc.)
- *LrgTweenBase* - Abstract base class for all tween types
- *LrgTween* - Single property animation
- *LrgTweenGroup* - Base class for compound tweens
- *LrgTweenSequence* - Run tweens one after another
- *LrgTweenParallel* - Run tweens simultaneously
- *LrgTweenManager* - Coordinates all active tweens

** Quick Start
:PROPERTIES:
:CUSTOM_ID: quick-start
:END:
#+begin_src C
/* Create a tween manager */
LrgTweenManager *manager = lrg_tween_manager_new ();

/* Create a simple opacity tween (target, property, duration) */
LrgTween *fade = lrg_tween_new (G_OBJECT (sprite), "opacity", 0.5f);
lrg_tween_set_from_float (fade, 0.0f);
lrg_tween_set_to_float (fade, 1.0f);
lrg_tween_base_set_easing (LRG_TWEEN_BASE (fade), LRG_EASING_EASE_OUT_CUBIC);

/* Add to manager and start */
lrg_tween_manager_add (manager, LRG_TWEEN_BASE (fade));
lrg_tween_base_start (LRG_TWEEN_BASE (fade));

/* In game loop */
lrg_tween_manager_update (manager, delta_time);
#+end_src

** Sequences and Parallel
:PROPERTIES:
:CUSTOM_ID: sequences-and-parallel
:END:
#+begin_src C
/* Create a sequence: fade in, wait, fade out */
LrgTweenSequence *seq = lrg_tween_sequence_new ();

LrgTween *fade_in = lrg_tween_new (G_OBJECT (label), "opacity", 0.3f);
lrg_tween_set_to_float (fade_in, 1.0f);

LrgTween *fade_out = lrg_tween_new (G_OBJECT (label), "opacity", 0.3f);
lrg_tween_set_to_float (fade_out, 0.0f);

lrg_tween_sequence_append (seq, LRG_TWEEN_BASE (fade_in));
lrg_tween_sequence_append_interval (seq, 1.0f);  /* 1 second delay */
lrg_tween_sequence_append (seq, LRG_TWEEN_BASE (fade_out));

/* Run multiple tweens at once */
LrgTweenParallel *parallel = lrg_tween_parallel_new ();
lrg_tween_parallel_add (parallel, LRG_TWEEN_BASE (move_tween));
lrg_tween_parallel_add (parallel, LRG_TWEEN_BASE (scale_tween));
lrg_tween_parallel_add (parallel, LRG_TWEEN_BASE (rotate_tween));
#+end_src

:CUSTOM_ID: looping
:END:
#+begin_src C
/* Infinite ping-pong loop */
lrg_tween_base_set_loop_count (LRG_TWEEN_BASE (tween), -1);  /* -1 = infinite */
lrg_tween_base_set_loop_mode (LRG_TWEEN_BASE (tween), LRG_TWEEN_LOOP_PING_PONG);

/* Fixed number of restarts */
lrg_tween_base_set_loop_count (LRG_TWEEN_BASE (tween), 3);
lrg_tween_base_set_loop_mode (LRG_TWEEN_BASE (tween), LRG_TWEEN_LOOP_RESTART);
#+end_src

** Headless / offline rendering with easing
:PROPERTIES:
:CUSTOM_ID: headless-offline-rendering
:END:
=LrgTweenManager= is designed for live, clock-driven use only — it has no
seek, rewind, or sample-at-time API.  For offline / headless work (baking
frames to a =GrlImage= sequence, writing a =GrlGifWriter=, procedural asset
generation) you drive time yourself with a plain loop.

*** Two-value case — use =lrg_easing_interpolate= directly

When you need to animate a single value between two endpoints, use the pure
easing functions without creating any tween object:

#+begin_src C
#define N_FRAMES 60

for (gint frame = 0; frame < N_FRAMES; frame++)
{
    gfloat t = (gfloat) frame / (gfloat)(N_FRAMES - 1); /* loop IS the clock */
    gfloat x = lrg_easing_interpolate (LRG_EASING_EASE_IN_OUT_CUBIC,
                                        x0, x1, t);
    /* draw using x into a GrlImage, append to a GrlGifWriter, etc. */
}
#+end_src

The loop variable is the only clock; no =LrgTweenManager= or =g_get_monotonic_time()=
is required.

*** Multi-key case — use =LrgKeyframeCurve=

For animations that pass through more than two values, each segment with its
own easing, use =LrgKeyframeCurve=.  It is a pure, clock-free =GObject= that
stores keyframes sorted by normalized time and samples them on demand:

#+begin_src C
#define N_FRAMES 90

g_autoptr (LrgKeyframeCurve) curve = lrg_keyframe_curve_new ();

/* Three keys: start at 0, peak at 80 (ease-in), land at 0 (ease-out) */
lrg_keyframe_curve_add_key (curve, 0.0f,  0.0f, LRG_EASING_EASE_IN_CUBIC);
lrg_keyframe_curve_add_key (curve, 0.5f, 80.0f, LRG_EASING_EASE_OUT_CUBIC);
lrg_keyframe_curve_add_key (curve, 1.0f,  0.0f, LRG_EASING_LINEAR);

for (gint frame = 0; frame < N_FRAMES; frame++)
{
    gfloat t = (gfloat) frame / (gfloat)(N_FRAMES - 1);
    gfloat y = lrg_keyframe_curve_sample (curve, t);
    /* draw sprite at (x, y) into GrlImage, then append frame ... */
}
#+end_src

Behaviour contracts for =lrg_keyframe_curve_sample()=:
- Zero keys: returns 0.0 (with a g_return_val_if_fail warning).
- One key: returns that key's value for all =t=.
- =t= before the first key: clamped to the first key's value.
- =t= after the last key: clamped to the last key's value.
- =t= outside [0,1] follows the same clamping rules.
- Duplicate-=t= keys: last =add_key= call at that time wins.
