** Building
:PROPERTIES:
:CUSTOM_ID: building
:END:
*** Basic Build
:PROPERTIES:
:CUSTOM_ID: basic-build
:END:
#+begin_src sh
make
#+end_src

This will:

1. Build submodule dependencies (graylib, yaml-glib)
2. Compile the Libregnum library (static and shared)
3. Generate GObject Introspection files (.gir and .typelib)

*** Debug Build
:PROPERTIES:
:CUSTOM_ID: debug-build
:END:
#+begin_src sh
make DEBUG=1
#+end_src

Enables debug symbols (=-g3=) and disables optimization (=-O0=).

*** Build Targets
:PROPERTIES:
:CUSTOM_ID: build-targets
:END:
| Target           | Description                     |
|------------------+---------------------------------|
| =make=           | Build library (default)         |
| =make lib=       | Build library only              |
| =make gir=       | Generate GIR files              |
| =make test=      | Build and run tests             |
| =make test-sanitize= | Run tests under ASan+UBSan (own build dir) |
| =make test-valgrind= | Run tests under valgrind (excludes scripting tests) |
| =make examples=  | Build examples                  |
| =make docs=      | Build documentation             |
| =make install=   | Install to PREFIX               |
| =make clean=     | Clean build artifacts           |
| =make distclean= | Clean everything including deps |
| =make help=      | Show all targets                |

#+end_src

*** Debug Options
:PROPERTIES:
:CUSTOM_ID: debug-options
:END:
#+begin_src makefile
DEBUG := 0             # Set to 1 for debug build
ENABLE_TRACE := 0      # Set to 1 for trace logging
SANITIZE := 0          # Set to 1 for ASan+UBSan build (implies DEBUG=1;
                       # builds into build/sanitize/ so it cannot collide
                       # with a normal debug build)
ASAN := 0              # AddressSanitizer (requires DEBUG=1)
UBSAN := 0             # UndefinedBehaviorSanitizer (requires DEBUG=1)
#+end_src

*** Testing & Memory Checking
:PROPERTIES:
:CUSTOM_ID: testing-memory-checking
:END:

Beyond =make test=, two targets verify memory correctness across the test
suite. Both set =G_SLICE=always-malloc= and =G_DEBUG=gc-friendly= at runtime
so GLib's slice allocator does not hide individual allocations from the tools
(=g_slice_*= is used across the codebase).

**** =make test-sanitize=

Builds the library and tests into an isolated =build/sanitize/= directory
with =-fsanitize=address,undefined= (=SANITIZE=1= implies =DEBUG=1=,
=ASAN=1=, =UBSAN=1=), runs the full suite with =detect_leaks=1=, and reports
any ASan/UBSan finding. Because the output directory is separate, no prior
=make clean= is needed and a normal =make test= run is unaffected.

#+begin_src sh
make test-sanitize
#+end_src

Confirmed-benign leaks (process-lifetime singletons such as
=LrgCrashReporter=) are filtered via [[file:../tests/lsan.supp][tests/lsan.supp]],
wired through =LSAN_OPTIONS=suppressions==. Leak suppressions go in
=LSAN_OPTIONS= (the =leak:<regex>= format), *not* =ASAN_OPTIONS= -- ASan's
=suppressions== accepts only interceptor types and rejects =leak:=. The
=test-scripting*= binaries are excluded from the sanitize run by default
(see =SANITIZE_BINARIES= in =tests/Makefile=) for the same reason as valgrind.

**** =make test-valgrind=

Builds a plain (non-instrumented) =DEBUG=1= build and runs the suite under
=valgrind --leak-check=full= with the starter suppressions file
[[file:../tests/valgrind.supp][tests/valgrind.supp]]. ASan and valgrind
conflict, so this target does *not* use sanitizers.

#+begin_src sh
make test-valgrind
#+end_src

The five =test-scripting*= binaries are excluded from the valgrind run by
default (see =VALGRIND_BINARIES= in =tests/Makefile=) because they =dlopen=
CPython/GJS, whose runtimes emit large benign leak footprints. Override with
=VALGRIND_TESTS=...= to run a custom set.

Any ASan/UBSan report or valgrind "definite leak" is a *real* defect to file,
not a tooling artifact. Only add a suppression to =tests/valgrind.supp= once
the leak is confirmed to be benign library/tool noise; the starter file is
expected to grow as real runs surface noise.

:CUSTOM_ID: installation
:END:
*** System Installation
:PROPERTIES:
:CUSTOM_ID: system-installation
:END:
#+begin_src sh

#+end_src

Default installation locations:

- Libraries: =/usr/local/lib/=
- Headers: =/usr/local/include/libregnum/=
- pkg-config: =/usr/local/lib/pkgconfig/=
- GIR: =/usr/local/share/gir-1.0/=
- Typelib: =/usr/local/lib/girepository-1.0/=

** Using Libregnum
:PROPERTIES:
:CUSTOM_ID: using-libregnum
:END:
*** pkg-config
:PROPERTIES:
:CUSTOM_ID: pkg-config
:END:
After installation:

#+begin_src sh
# Compiler flags
pkg-config --cflags libregnum-1

# Linker flags
pkg-config --libs libregnum-1
#+end_src

*** In a Makefile
:PROPERTIES:
:CUSTOM_ID: in-a-makefile
:END:
#+begin_src makefile
CFLAGS += $(shell pkg-config --cflags libregnum-1)
LDFLAGS += $(shell pkg-config --libs libregnum-1)
#+end_src

*** Without Installation
:PROPERTIES:
:CUSTOM_ID: without-installation
:END:
Set library paths before running:

#+begin_src sh
export LD_LIBRARY_PATH="/path/to/libregnum/build/release/lib:$LD_LIBRARY_PATH"
export PKG_CONFIG_PATH="/path/to/libregnum/build/release:$PKG_CONFIG_PATH"
export GI_TYPELIB_PATH="/path/to/libregnum/build/release/gir:$GI_TYPELIB_PATH"
#+end_src

** Testing
:PROPERTIES:
:CUSTOM_ID: testing
:END:
Run the test suite:

#+begin_src sh
make test
#+end_src

This builds and runs all unit tests using the GLib testing framework.

To run tests manually:

#+begin_src sh
cd tests
LD_LIBRARY_PATH="../build/release/lib" ./test-engine --tap
#+end_src

** Build System Details
:PROPERTIES:
:CUSTOM_ID: build-system-details
:END:
Libregnum uses GNU Make with a three-file structure:

- *Makefile* - Main build orchestration
- *config.mk* - Configuration variables
- *rules.mk* - Build rules and helper functions

The build system is modeled after graylib's proven pattern.
